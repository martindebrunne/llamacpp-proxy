# Proxy Logs

Structured logs generated by the proxy server.

## Log Format

File logs are written as **one JSON object per line**.

### Entry Types
- `REQUEST_START` - Request metadata + request payload
- `REQUEST_END` - Final status, duration, response size + response payload
- `STREAM_CHUNK` - Streaming chunk details (for streamed responses)
- `INFO` / `ERROR` - Internal logger messages

### Example Entries
```json
{"timestamp":"2026-10-04 04:54:01.123","type":"REQUEST_START","correlationId":"abc123","method":"POST","path":"/v1/chat/completions","stream":false,"incomingModel":"MyModel-Think","upstreamModel":null,"thinkingMode":null,"requestPayload":{"model":"MyModel-Think","messages":[{"role":"user","content":"Hello"}]}}
{"timestamp":"2026-10-04 04:54:02.357","type":"REQUEST_END","correlationId":"abc123","status":200,"duration":1234,"responseSize":2048,"stream":false,"upstreamModel":"MyModel","thinkingMode":"true","responsePayload":{"choices":[{"message":{"content":"Hello there!"}}],"usage":{"prompt_tokens":10,"completion_tokens":5,"total_tokens":15}}}
```

## Console Output

When running the proxy, console logs use a compact two-phase format:

```text
[12:00:00] REQ_IN  POST  /v1/chat/completions | model=MyModel-Think | thinking=- | correlation=abc123
[12:00:01] REQ_OUT POST  /v1/chat/completions | status=200 | duration=850ms | size=1.8KB | upstream=MyModel | thinking=true | stream=true | correlation=abc123
```

## Running the Proxy

```bash
# Default ports (proxy: 4000, upstream: 8080)
npm start

# Custom proxy/upstream ports via CLI arg
npm start -- 3000:8081
```

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `PROXY_HOST` | `127.0.0.1` | Host for the proxy server |
| `PROXY_PORT` | `4000` | Port for the proxy server (used if no CLI override) |
| `UPSTREAM_HOST` | `127.0.0.1` | Host for upstream llama.cpp server |
| `UPSTREAM_PORT` | `8080` | Port for upstream llama.cpp server (used if no CLI override) |

## Log Rotation

Logs are automatically rotated:
- **Time-based**: rotation every 24h
- **Size-based**: rotation when file exceeds **5MB**

## Location

Logs are stored in `logs/` as `proxy-*.log`.

## Viewing Logs

```bash
# Follow latest log entries
tail -f logs/proxy-*.log

# Filter error entries
grep '"type":"ERROR"' logs/proxy-*.log

# Filter by correlation ID
grep 'abc123' logs/proxy-*.log
```
