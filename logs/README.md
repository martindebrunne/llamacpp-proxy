# Proxy Logs

Human-readable log files generated by the proxy server.

## Log Format

Each log entry follows this format:

```
[YYYY-MM-DD HH:MM:SS.mmm] LEVEL message | key=value | key=value
```

### Log Levels
- `INFO` - Informational messages (startup, shutdown)
- `ERROR` - Error messages
- `REQUEST` - Request/response logs

### Request Log Format
```
[2026-07-04 17:10:00.123] REQUEST POST /chat/completions | method=POST | path=/chat/completions | incoming=Qwen-Think | upstream=Qwen3.5-35B-A3B-T | thinking=true | status=200 | duration=1234ms | request={...full payload...} | response={...full payload...}
```

### Console Output Format
When running `node proxy.js`, you'll see compressed output:
```
[17:58:26] REQ   POST  /chat/completions | model=Qwen3.5-35B-A3B -> Qwen3.5-35B-A3B | thinking=âś“ | status=âś“200 | duration=2495ms
```

## Configuration

### npm Scripts

```bash
# Use default ports (proxy: 4000, upstream: 8080)
npm start

# Custom proxy and upstream ports
npm start 4000:8080
npm start 3000:8081
```

### Command Line Arguments

```bash
# Use default ports (proxy: 4000, upstream: 8080)
node proxy.js

# Custom proxy and upstream ports
node proxy.js 3000:8081
```

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `PROXY_HOST` | `127.0.0.1` | Host for the proxy server |
| `PROXY_PORT` | `4000` | Port for the proxy server (overridden by command line) |
| `UPSTREAM_HOST` | `127.0.0.1` | Host for the upstream llama.cpp server |
| `UPSTREAM_PORT` | `8080` | Port for the upstream llama.cpp server (overridden by command line) |

### Usage Examples

```bash
# Use default ports
npm start
node proxy.js

# Custom proxy port only
PROXY_PORT=3000 npm start
PROXY_PORT=3000 node proxy.js

# Custom upstream port only
UPSTREAM_PORT=8081 npm start
UPSTREAM_PORT=8081 node proxy.js

# Custom both ports via environment
PROXY_PORT=3000 UPSTREAM_PORT=8081 npm start
PROXY_PORT=3000 UPSTREAM_PORT=8081 node proxy.js

# Custom both ports via npm start
npm start 3000:8081

# Custom both ports via command line
node proxy.js 3000:8081

# Custom host and ports
PROXY_HOST=0.0.0.0 PROXY_PORT=4000 UPSTREAM_HOST=127.0.0.1 UPSTREAM_PORT=8080 npm start
PROXY_HOST=0.0.0.0 PROXY_PORT=4000 UPSTREAM_HOST=127.0.0.1 UPSTREAM_PORT=8080 node proxy.js
```

## Log Rotation

Logs are automatically rotated:
- **Time-based**: New file each day (`proxy-YYYY-MM-DD.log`)
- **Size-based**: New file when current exceeds 10MB

## Location

Logs are stored in the `logs/` directory relative to the proxy server.

## Viewing Logs

```bash
# View current day's logs
cat logs/proxy-$(date +%Y-%m-%d).log

# Follow logs in real-time
tail -f logs/proxy-$(date +%Y-%m-%d).log

# Search for errors
grep ERROR logs/proxy-*.log

# Search for specific model
grep "Qwen-Think" logs/proxy-*.log

# View logs
cat logs/proxy-$(date +%Y-%m-%d).log
```

## Fields

| Field | Description |
|-------|-------------|
| `method` | HTTP method (GET, POST, etc.) |
| `path` | Request path |
| `incoming` | Model name sent by client |
| `upstream` | Model name sent to upstream |
| `thinking` | Thinking mode enabled (true/false) |
| `status` | HTTP status code |
| `duration` | Request duration in milliseconds |
| `request` | Full request JSON body |
| `response` | Full response JSON body |

## Console vs File Logging

- **Console**: Compressed format with thinking mode indicator (âś“/âś—), status indicator (âś“200/âś—500), no payloads
- **File**: Full format with complete request/response payloads (no truncation)
